## ❓ Frequently Asked Questions

**Q: Can I start implementing without Figma designs?**  
A: No. Get Figma designs first, compare with DESIGN_SYSTEM_SPEC.md, then proceed.

**Q: Which page should I start with?**  
A: Start with Scan page (P0) - highest impact, mobile-only, clearest requirements.

**Q: Do I need to update all 50+ CSS files?**  
A: No. Update CSS to use design tokens, but only redesign the 10 P0-P2 pages for now.

**Q: Should I use Tailwind CSS?**  
A: No. Project uses CSS Modules. Keep existing approach, just improve it with tokens.

**Q: What about dark mode?**  
A: Only if Figma includes dark mode designs. Currently partially implemented (not required).

**Q: Can I add new packages (UI library, etc.)?**  
A: No. Stay with existing stack: CSS Modules, Lucide React, no additional dependencies.

**Q: How do I know if I'm done?**  
A: When all P0-P2 pages match Figma and pass QA checklist in AUDIT_REPORT.md section 12.

**Q: Where do I find component usage examples?**  
A: COMPONENT_MANIFEST.md has all 70+ components with props and usage details.

**Q: What if Figma has different colors than the audit?**  
A: Trust Figma. Update design tokens accordingly. Document differences for future reference.
